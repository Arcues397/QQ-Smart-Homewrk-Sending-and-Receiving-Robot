电工五班作业收发机器人

A Python-based QQ bot for automated homework collection, packaging, and secure return distribution.

====================================

项目简介

本项目基于 QQ 机器人，实现班级作业的自动收集、打包导出、老师返还文件导入以及学生按身份查询返还文件。

解决的问题：
- 文件命名混乱
- 批量整理困难
- 老师返还 zip 难以分发
- 学生不能看到他人文件
- 已交/缺交统计麻烦

====================================

功能特性

学生端：
- 提交作业信息
- 上传 PDF
- 学号姓名自动校验
- 自动命名与归档
- 查询并领取返还文件

管理员端：
- 打包导出作业
- 导入老师返还 zip
- 自动匹配返还文件
- 导出已交名单
- 导出缺交名单
- 查看未匹配文件
- 手动匹配文件

系统特性：
- 支持多课程、多批次
- 支持不同命名模板
- 自动记录提交历史
- 支持重复提交覆盖
- 基于 openid 的身份隔离

- Python

====================================

环境要求

Python 3.10+

安装依赖：

pip install qq-botpy python-dotenv httpx pandas openpyxl

====================================

配置文件

创建 .env：

BOT_APPID=你的AppID
BOT_APPSECRET=你的AppSecret
BOT_USE_SANDBOX=true
ADMIN_OPENID=你的openid

====================================

学生使用方法

1. 提交作业：

提交 <类别> <批次> <学号> <姓名>

例如：
提交 大物实验 实验2 2025xxxxxxxxxx 张三

2. 然后发送 PDF 文件

3. 查询返还：

查询 <类别> <批次>

====================================

管理员命令

打包：
/打包 <类别> <批次>

导入返还：
/导入返还 <类别> <批次>

已交：
/已交 <类别> <批次>

缺交：
/缺交 <类别> <批次>

未匹配：
/未匹配 <类别> <批次>

手动匹配：
/手动匹配 <类别> <批次> <学号> <文件名>

====================================

核心流程

学生提交：
提交信息 → 校验 → 上传PDF → 自动命名 → 归档 → 记录

老师返还：
导入zip → 解压 → 自动匹配 → 存储 → 学生查询领取

====================================

数据文件

submissions.csv：提交记录
bindings.csv：身份绑定
returns.csv：返还记录

====================================

安全机制

- 学生只能获取自己的文件
- 使用 openid 自动识别身份
- 管理员权限受限

====================================

部署建议

推荐：
- 腾讯云轻量服务器
- Ubuntu 24.04
- 使用 systemd 常驻运行

====================================

注意事项

- 必须先发提交信息再发 PDF
- 学生首次提交后才建立绑定
- 未匹配文件可手动处理
- 文件命名越规范，匹配越准确
